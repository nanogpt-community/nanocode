The NanoCode CLI by default starts the [TUI](/docs/tui) when run without any arguments.

```bash
nanocode
```

But it also accepts commands as documented on this page. This allows you to interact with NanoCode programmatically.

```bash
nanocode run "Explain how closures work in JavaScript"
```

---

### tui

Start the NanoCode terminal user interface.

```bash
nanocode [project]
```

#### Flags

| Flag         | Short | Description                                |
| ------------ | ----- | ------------------------------------------ |
| `--continue` | `-c`  | Continue the last session                  |
| `--session`  | `-s`  | Session ID to continue                     |
| `--prompt`   |       | Prompt to use                              |
| `--model`    | `-m`  | Model to use in the form of provider/model |
| `--agent`    |       | Agent to use                               |
| `--port`     |       | Port to listen on                          |
| `--hostname` |       | Hostname to listen on                      |

## Commands

The NanoCode CLI also has the following commands.

---

### agent

Manage agents for NanoCode.

```bash
nanocode agent [command]
```

---

### attach

Attach a terminal to an already running NanoCode backend server started via `serve` or `web` commands.

```bash
nanocode attach [url]
```

This allows using the TUI with a remote NanoCode backend. For example:

```bash
# Start the backend server for web/mobile access
nanocode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
nanocode attach http://10.20.30.40:4096
```

#### Flags

| Flag        | Short | Description                       |
| ----------- | ----- | --------------------------------- |
| `--dir`     |       | Working directory to start TUI in |
| `--session` | `-s`  | Session ID to continue            |

---

#### create

Create a new agent with custom configuration.

```bash
nanocode agent create
```

This command will guide you through creating a new agent with a custom system prompt and tool configuration.

---

#### list

List all available agents.

```bash
nanocode agent list
```

---

### auth

Command to manage credentials and login for providers.

```bash
nanocode auth [command]
```

---

#### login

NanoCode is powered by the provider list at [Models.dev](https://models.dev), so you can use `nanocode auth login` to configure API keys for any provider you'd like to use. This is stored in `~/.local/share/nanocode/auth.json`.

```bash
nanocode auth login
```

When NanoCode starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a `.env` file in your project.

---

#### list

Lists all the authenticated providers as stored in the credentials file.

```bash
nanocode auth list
```

Or the short version.

```bash
nanocode auth ls
```

---

#### logout

Logs you out of a provider by clearing it from the credentials file.

```bash
nanocode auth logout
```

---

### github

Manage the GitHub agent for repository automation.

```bash
nanocode github [command]
```

---

#### install

Install the GitHub agent in your repository.

```bash
nanocode github install
```

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. [Learn more](/docs/github).

---

#### run

Run the GitHub agent. This is typically used in GitHub Actions.

```bash
nanocode github run
```

##### Flags

| Flag      | Description                            |
| --------- | -------------------------------------- |
| `--event` | GitHub mock event to run the agent for |
| `--token` | GitHub personal access token           |

### mcp

Manage Model Context Protocol servers.

```bash
nanocode mcp [command]
```

---

#### add

Add an MCP server to your configuration.

```bash
nanocode mcp add
```

This command will guide you through adding either a local or remote MCP server.

---

#### list

List all configured MCP servers and their connection status.

```bash
nanocode mcp list
```

Or use the short version.

```bash
nanocode mcp ls
```

---

#### auth

Authenticate with an OAuth-enabled MCP server.

```bash
nanocode mcp auth [name]
```

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

```bash
nanocode mcp auth list
```

Or use the short version.

```bash
nanocode mcp auth ls
```

---

#### logout

Remove OAuth credentials for an MCP server.

```bash
nanocode mcp logout [name]
```

---

#### debug

Debug OAuth connection issues for an MCP server.

```bash
nanocode mcp debug <name>
```

### models

List all available models from configured providers.

```bash
nanocode models [provider]
```

This command displays all models available across your configured providers in the format `provider/model`.

This is useful for figuring out the exact model name to use in [your config](/docs/config/).

You can optionally pass a provider ID to filter models by that provider.

```bash
nanocode models anthropic
```

#### Flags

| Flag        | Description                                                  |
| ----------- | ------------------------------------------------------------ |
| `--refresh` | Refresh the models cache from models.dev                     |
| `--verbose` | Use more verbose model output (includes metadata like costs) |

Use the `--refresh` flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in NanoCode.

```bash
nanocode models --refresh
```

### run

Run nanocode in non-interactive mode by passing a prompt directly.

```bash
nanocode run [message..]
```

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

```bash "nanocode run"
nanocode run Explain the use of context in Go
```

You can also attach to a running `nanocode serve` instance to avoid MCP server cold boot times on every run:

```bash
# Start a headless server in one terminal
nanocode serve

# In another terminal, run commands that attach to it
nanocode run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flags

| Flag         | Short | Description                                                        |
| ------------ | ----- | ------------------------------------------------------------------ |
| `--command`  |       | The command to run, use message for args                           |
| `--continue` | `-c`  | Continue the last session                                          |
| `--session`  | `-s`  | Session ID to continue                                             |
| `--share`    |       | Share the session                                                  |
| `--model`    | `-m`  | Model to use in the form of provider/model                         |
| `--agent`    |       | Agent to use                                                       |
| `--file`     | `-f`  | File(s) to attach to message                                       |
| `--format`   |       | Format: default (formatted) or json (raw JSON events)              |
| `--title`    |       | Title for the session (uses truncated prompt if no value provided) |
| `--attach`   |       | Attach to a running nanocode server (e.g., http://localhost:4096)  |
| `--port`     |       | Port for the local server (defaults to random port)                |

### serve

Start a headless NanoCode server for API access. Check out the [server docs](/docs/server) for the full HTTP interface.

```bash
nanocode serve
```

This starts an HTTP server that provides API access to nanocode functionality without the TUI interface. Set `NANOGPT_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `nanocode`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

```bash
nanocode session [command]
```

---

#### list

List all NanoCode sessions.

```bash
nanocode session list
```

##### Flags

| Flag          | Short | Description                          |
| ------------- | ----- | ------------------------------------ |
| `--max-count` | `-n`  | Limit to N most recent sessions      |
| `--format`    |       | Output format: table or json (table) |

---

### stats

Show token usage and cost statistics for your NanoCode sessions.

```bash
nanocode stats
```

#### Flags

| Flag        | Description                                                                 |
| ----------- | --------------------------------------------------------------------------- |
| `--days`    | Show stats for the last N days (all time)                                   |
| `--tools`   | Number of tools to show (all)                                               |
| `--models`  | Show model usage breakdown (hidden by default). Pass a number to show top N |
| `--project` | Filter by project (all projects, empty string: current project)             |

---

### export

Export session data as JSON.

```bash
nanocode export [sessionID]
```

If you don't provide a session ID, you'll be prompted to select from available sessions.

---

### import

Import session data from a JSON file or NanoCode share URL.

```bash
nanocode import <file>
```

You can import from a local file or an NanoCode share URL.

```bash
nanocode import session.json
nanocode import https://opncd.ai/s/abc123
```

---

### web

Start a headless NanoCode server with a web interface.

```bash
nanocode web
```

This starts an HTTP server and opens a web browser to access nanocode through a web interface. Set `NANOGPT_SERVER_PASSWORD` to enable HTTP basic auth (username defaults to `nanocode`).

#### Flags

| Flag         | Description                                |
| ------------ | ------------------------------------------ |
| `--port`     | Port to listen on                          |
| `--hostname` | Hostname to listen on                      |
| `--mdns`     | Enable mDNS discovery                      |
| `--cors`     | Additional browser origin(s) to allow CORS |

---

### acp

Start an ACP (Agent Client Protocol) server.

```bash
nanocode acp
```

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

#### Flags

| Flag         | Description           |
| ------------ | --------------------- |
| `--cwd`      | Working directory     |
| `--port`     | Port to listen on     |
| `--hostname` | Hostname to listen on |

---

### uninstall

Uninstall NanoCode and remove all related files.

```bash
nanocode uninstall
```

#### Flags

| Flag            | Short | Description                                 |
| --------------- | ----- | ------------------------------------------- |
| `--keep-config` | `-c`  | Keep configuration files                    |
| `--keep-data`   | `-d`  | Keep session data and snapshots             |
| `--dry-run`     |       | Show what would be removed without removing |
| `--force`       | `-f`  | Skip confirmation prompts                   |

---

### upgrade

Updates nanocode to the latest version or a specific version.

```bash
nanocode upgrade [target]
```

To upgrade to the latest version.

```bash
nanocode upgrade
```

To upgrade to a specific version.

```bash
nanocode upgrade v0.1.48
```

#### Flags

| Flag       | Short | Description                                                       |
| ---------- | ----- | ----------------------------------------------------------------- |
| `--method` | `-m`  | The installation method that was used; curl, npm, pnpm, bun, brew |

## Environment variables

NanoCode can be configured using environment variables.

| Variable                              | Type    | Description                                                |
| ------------------------------------- | ------- | ---------------------------------------------------------- |
| `NANOGPT_AUTO_SHARE`                  | boolean | Automatically share sessions                               |
| `NANOGPT_GIT_BASH_PATH`               | string  | Path to Git Bash executable on Windows                     |
| `NANOGPT_CONFIG`                      | string  | Path to config file                                        |
| `NANOGPT_CONFIG_DIR`                  | string  | Path to config directory                                   |
| `NANOGPT_CONFIG_CONTENT`              | string  | Inline json config content                                 |
| `NANOGPT_DISABLE_AUTOUPDATE`          | boolean | Disable automatic update checks                            |
| `NANOGPT_DISABLE_PRUNE`               | boolean | Disable pruning of old data                                |
| `NANOGPT_DISABLE_TERMINAL_TITLE`      | boolean | Disable automatic terminal title updates                   |
| `NANOGPT_PERMISSION`                  | string  | Inlined json permissions config                            |
| `NANOGPT_DISABLE_DEFAULT_PLUGINS`     | boolean | Disable default plugins                                    |
| `NANOGPT_DISABLE_LSP_DOWNLOAD`        | boolean | Disable automatic LSP server downloads                     |
| `NANOGPT_ENABLE_EXPERIMENTAL_MODELS`  | boolean | Enable experimental models                                 |
| `NANOGPT_DISABLE_AUTOCOMPACT`         | boolean | Disable automatic context compaction                       |
| `NANOGPT_DISABLE_CLAUDE_CODE`         | boolean | Disable reading from `.claude` (prompt + skills)           |
| `NANOGPT_DISABLE_CLAUDE_CODE_PROMPT`  | boolean | Disable reading `~/.claude/CLAUDE.md`                      |
| `NANOGPT_DISABLE_CLAUDE_CODE_SKILLS`  | boolean | Disable loading `.claude/skills`                           |
| `NANOGPT_DISABLE_EXTERNAL_SKILLS`     | boolean | Disable loading external `.claude`/`.agents` skills        |
| `NANOGPT_DISABLE_MODELS_FETCH`        | boolean | Disable fetching models from remote sources                |
| `NANOGPT_FAKE_VCS`                    | string  | Fake VCS provider for testing purposes                     |
| `NANOGPT_DISABLE_FILETIME_CHECK`      | boolean | Disable file time checking for optimization                |
| `NANOGPT_CLIENT`                      | string  | Client identifier (defaults to `cli`)                      |
| `NANOGPT_ENABLE_EXA`                  | boolean | Enable Exa web search tools                                |
| `NANOGPT_SERVER_PASSWORD`             | string  | Enable basic auth for `serve`/`web`                        |
| `NANOGPT_SERVER_USERNAME`             | string  | Override basic auth username (default `nanocode`)          |
| `NANOGPT_MODELS_URL`                  | string  | Custom URL for fetching models configuration               |
| `NANOGPT_MODELS_PATH`                 | string  | Local path for models configuration override               |
